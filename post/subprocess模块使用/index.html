推荐使用python>=3.6 , subprocess.run()

subprocess run()功能在python3.5被添加，为官方推荐使用的子流程调用方法

subprocess.run(args, *, stdin=None, input=None, stdout=None, stderr=None, capture_output=False, shell=False, cwd=None, timeout=None, check=False, encoding=None, errors=None, text=None, env=None, universal_newlines=None)

注: 以上参数不是在所有版本都适用的,比如capture_output在python3.7才被添加

    import subprocess
    try:
        listl = subprocess.run('ls -l',shell=True,check=True,cwd='/home/chengjiawen1/software',stdout=subprocess.PIPE,stderr=open('eee.e','w'),encoding='utf-8',timeout=1)
    except subprocess.CalledProcessError as err:
        print('ERROR:',err)
    else:
        print('returncode:',listl.returncode) #listl.returncode获得程序运行的退出状态码
        print('CMD stdout:',listl.stdout) #listl.stdout获得程序运行的标准输出
    #输出结果：
    returncode: 0
    CMD stdout: 'total 47136\ndrwxr-xr-x  6 chengjiawen1 bc_ccp      101 Aug 26 17:53 cgmaptools-0.1.2\ndrwxr-xr-x 11 chengjiawen1 bc_ccp     4096 Aug 22 21:24 lrzsz-0.12.20\n

示例参数说明：

  • shell=True 子进程生成一个中间shell进程然后运行该命令，默认情况shell=False直接运行命令；当第一个命令参数为字符串时需要设置shell=True
  • check=True 命令运行返回状态码非0，抛出CalledProcessError
  • cwd 指的命令工作路径
  • stdout 命令标准输出，可以输出到文件也可以为subprocess.PIPE[方便后续调用命令输出结果]
  • stderr 命令标准错误输出， 可以输出到文件也可以为subprocess.PIPE
  • encodeing=‘utf-8’ 指的输出编码方式[这个官方文档解释的清楚]
  • timeout=1 指定命令timeout时间，单位为秒，超时会抛出TimeoutExpired异常

subprocess.run的其他参数说明：

  • capture_output=True 将会自动创建stdout=subprocess.PIPE和stderr=subprocess.PIPE，如果希望捕获合并标准输出与错误输出可以使用stdout=subprocess.PIPE和stderr=subprocess.STDOUT 【python3.7新增】

>>> wcc = subprocess.Popen(['wc','-c'],stdin=subprocess.PIPE,stdout=subprocess.PIPE)
>>> hhh = 'Hello oocarain'.encode('utf-8')
>>> wordbyte = wcc.communicate(hhh)[0].decode('utf-8')
>>> wordbyte
'14\n'

使用run方法更为简单:
>>> b = subprocess.run(['wc','-c'],check=True,input="Hello oocarain",stdout=subprocess.PIPE,stderr=subprocess.PIPE,encoding='utf-8',timeout=1)
>>> b
CompletedProcess(args=['wc', '-c'], returncode=0, stdout='14\n', stderr='')
#
#注 input输入参数被传递给Popen.communicate(),然后传递给子进程的stdin,并且内部会自动创建stdin=subprocess.PIPE,所以input和stdin两个参数不能同时使用

管道链接

通过创建单独的Popen实例并将他们的stdin和stdout链接在一起,起到类似于linux shell的管道命令运算

>>> cat = subprocess.Popen(['cat','a.yaml'],stdout=subprocess.PIPE)
>>> grep = subprocess.Popen(['grep','hello'],stdin=cat.stdout,stdout=subprocess.PIPE)
>>> cut = subprocess.Popen(['awk','-F',':','{print $2}'],stdin=grep.stdout,stdout=subprocess.PIPE)
>>> end_of_pipe = cut.stdout
>>> print(end_of_pipe.readline().decode('utf-8'))
 chengjiawen #hello

# 以上操作相当于相当于: cat a.yaml | grep hello | awk -F : {print $2}
